About

For the moment, I've only created a static version of the application, the source are on GitHub under the app folder. This application is responsive.

Development

I've used Yeoman to create the structure of the application.
But I've put images and css under a folder named assets
Yeoman come with Grunt, so multiple tasks are automatic :
Compilation of the sass files, Live reload is active, so you don't need to reload the browser to see the modifications after writing js or css
The js files are watched with jshint, to get informed about the errors.

Build

When you build your application, the js files are concatened and minified, to increase the performance.
The css file is also minified and there is a .htaccess file (from the html5 boilerplate) to do some optimisation, and serve the files gzipped.
This html5 boilerplate is used by Yeoman to create the index.html page.

The build phase is also the moment where you run the javascript test, Yeoman come with Karma.

CSS

The css file is create with a framework named inuit.css
Unlike Bootstrap, there's no style inside, so you have to create the design that you want. The css file is versioned with grunt at the build, so this file is considered as new, and not served from the cache.

The inuit.css is very versatile, you can download (with bower) only the files that you needs, to serve the application without extra code.
I've added my own code inside a folder named ui. I use the same structure than inuit, base.xxx for the base files (links, images, etc), objects.xxx, components.xxx, settings.xxx for my variables files (theses variables are great to design easily different design of a application/website), some tools and widths.
The goal is to write OOCSS components, with BEM notation.
So, we have a great base to write application/website by adding modules.
For theses pages, the size of the css is only 3,6KB after compression.

Icons

I use icomoon to create a font with the icon used in the application.
The .scss files is named components.icons-fonts and has the code for the 4 icons I use in the application.
If I need more icons, I have to go the website to generate a new font file. The goal is always to have the smallest assets to serve, to give to the customers the best experience.
